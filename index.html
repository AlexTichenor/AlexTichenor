The Purpose of the Mobile App

The purpose of the creating a mobile app for Heating Up DFS is be more efficient in meeting the user’s needs. Most of the current users of the website want to be able to use the application on the go and away from home. The owner wanted to be able to reach more potential users and continue to grow the business.

The useflow was create by looking at the existing website and mapping out a user journey according to the current navigation The userflow helped see how the user would navigate the app.

Heating Up DFS

The client requested a mobile app to compliment an existing fantasy sports website. The mobile app was designed could coexist with the already created website. By utilizing the same design features and elements of the existing website I was able to incoporate them into the mobile app’s designs. The app has NBA and NFL articles, fantasy projections, a calendar to see upcoming articles, and a podcast to hear about everything fantasy sports related.

QuikSplit

Split Quik makes splitting a restuarant bill with friends, coworkers, family, or whoever easy. Request to be paid, select who needs to pay you, type in the amount, and get paid for last night restuarant bill. With Split Quik you will not have to worry about paying extra on your bill. Split Quik will make sure your bill is divided evenly.

While developing Split Quik there were some problems I run into. The problems started when I saw that requesting money up from what not the best option. Instead user can send request for payment from their friends. This would make it easier for user to receive money after the fact. Then be able to track payment into the app seeing their bills and who has not paid.

Challenges Faced Along the Way

Misconceptions about what elements do and navigations. One of the biggest challenges l faced making elements readable and user friendly. Some of my early icons were misleading and confusing to my users in the user testing.

These challenges were overcomed by using universal icons that users are already familiar with instead of using unfamiliar ones. By taking the feedback I received from my users I was able to overcome some of the challenges in navigation that happened earlier in my design process.
